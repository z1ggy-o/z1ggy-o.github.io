如何让普通 markdown 和 hugo 统一管理图片的方式

Hugo 实际只支持两种方式来管理图片：

  1. 使用 page bundle 的形式，将 md 文件和对应的图片文件放在同一个目录下
  2. 将图片文件放在 static 目录下，并在 md 文件中引用

我的目标是将整个 content 当作当作单独可使用的文件库，同时想要能够正常使用 hugo 将其编译成网站。所以使用 page bundle 的形式不太好，需要 为每个需要用到图片的文件都再创建一个文件夹，麻烦。所以采用第二种方式。但问题是，static 目录是和 content 目录同级的，图片和文件就被分开了。

还好 git 是支持软链接的，我们可以将图片统一放在 content/attachments 目录下，然后在 static 下做一个软链接，将其指向 attachments。 写文的时候使用 ![image-name](/attachments/image-name.png) 的模式引用图片，则不论是单独使用 md 文件，还是编译成静态网站，都可以正常索引。

如何居中图片

在 assets/css/ 下创建 custom.css 文件，可载入自定义样式。有些主题会自动读取，有些可能需要手动在 config.toml 中开启。

# 配置使用 custom.css 文件，在 `/config/_default/params.toml` 中添加
 [params.assets]
   customCSS = ["css/custom.css"]
/* 居中，自动响应 */
.article-content img {
  display: block;
  margin: 0 auto;
  max-width: 1200px;
  max-height: 800px;
  border-radius: 2px;
}
